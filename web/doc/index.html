Compatibilité javascript cross-browser
Un moyen secure de code des appli js cross-browser est de n'utiliser que des fonctionnalité javascript standard, c'est à dire conforme à la version du standard ecmascript supporté par tous les browser.

Cette page wiki indique quel niveau de support de quelle version d'ecmascript est incluse dans les versions courante des implémentations javascript de ces browser.

A date, c'est donc la version "ECMA-262, edition 5.1" avec laquelle il faut produire son code javascript cross-browser. Si l'on veut supporter des ancienne version il vaut mieux passer par une librairie supportant les (certaines) anciennes version de browser comme jQuery ou autre. Les tuto et autres références javascript donnés dans ce doc doivent donc être conformes à ce standard. Ces tutos sont parfois moins clair que ce standard...Par exemple, pour savoir si "new Function(...)" est éqivalent à "Function(...)" dans une expression, les tutos sont moin clair que le standard qui dit tous simplement que oui : 
15.3.1 The Function Constructor Called as a Function

When Function is called as a function rather than as a constructor, it creates and initialises a new Function object. Thus the function call Function(…) is equivalent to the object creation expression new Function(…) with the same arguments.
